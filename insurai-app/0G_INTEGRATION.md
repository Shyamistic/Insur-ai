# InsurAI — 0G Modular Integration Architecture

InsurAI leverages the full **0G Ecosystem** to build a trustless, autonomous parametric insurance platform. This document outlines the technical integration points.

## 1. 0G Chain (Settlement Layer)
The core business logic resides in the `InsurancePolicy` smart contract deployed on the **0G Galileo Testnet**.

- **EVM Compatibility**: Built with Solidity 0.8.20 and deployed using the `cancun` EVM version.
- **Parametric Logic**: Policies are defined by objective parameters (e.g., flight number, device ID).
- **Funds Management**: Premiums and payouts are handled in native `$0G` tokens.
- **Verification**: The contract uses `ecrecover` to verify ECDSA signatures generated by the TEE enclave, ensuring that only AI-verified claims are paid out.

## 2. 0G Compute (Verification Layer)
Claims are processed using **0G Compute** in a Hardware-Isolated environment (TEE).

- **Intel TDX/SGX**: The AI inference model runs inside a secure enclave, ensuring that the evidence data (e.g., medical records, travel documents) is never exposed to the internet or the insurer.
- **Cryptographic Attestation**: Upon approving a claim, the enclave signs a message containing the `claimId`, `payoutAmount`, and `approvalStatus`.
- **Enclave Identity**: The contract stores the `teeSignerAddress`, which corresponds to the public key of the enclave's hardware-bound keypair.

## 3. 0G Storage (Data Layer)
Evidence and policy metadata are stored on **0G Storage** for immutable, high-performance data availability.

- **Content Addressing**: Every claim is associated with a Content Identifier (CID).
- **Data Availability**: CIDs are stored onchain in the `InsurancePolicy` contract, allowing anyone to verify that the evidence exists while keeping the actual data private or stored efficiently offchain.
- **Scaling**: By offloading heavy evidence files (images/PDFs) to 0G Storage, the system remains scalable even with millions of active policies.

## 🛠️ Developer Deployment Guide

### 1. Prerequisites
- Node.js & npm
- A wallet with 0G Testnet tokens (get them at [faucet.0g.ai](https://faucet.0g.ai))

### 2. Environment Setup
Create a `.env.local` file in the root:
```bash
DEPLOYER_PRIVATE_KEY=your_private_key_here
TEE_SIGNER_ADDRESS=your_enclave_public_address_here
NEXT_PUBLIC_CONTRACT_ADDRESS=deployed_address_after_step_3
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_id
```

### 3. Deploy Contract
```bash
# Install dependencies
npm install

# Compile contracts
npx hardhat compile

# Deploy to 0G Galileo
npx hardhat run scripts/deploy.ts --network 0g-galileo
```

### 4. Run Frontend
```bash
npm run dev
```
